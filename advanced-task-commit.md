# 고급업무커밋

고급업무 스킬로 수행한 작업 내역을 업무계획서 기반으로 정리하여 커밋하는 스킬입니다.

## 사용법

`/advanced-task-commit [파일명 또는 키워드]`

- 인자가 없으면: `.claude/tasks/` 폴더의 "진행중" 또는 최근 "완료" 업무를 자동 탐색
- 파일명/키워드 지정: 해당 업무계획서를 직접 로드

## 수행 절차

### Phase 1: 변경 내역 분석

1. **업무 문서 로드**: 업무계획서(`task_*.md`)와 TodoList(`todo_*.md`)를 읽습니다.
2. **Git 상태 확인**: `git status`와 `git diff`로 실제 변경된 파일을 확인합니다.
3. **변경 매핑**: 업무계획서의 "변경 사항 목록"과 실제 git 변경 사항을 대조합니다.
   - 계획에 있지만 변경되지 않은 파일 식별
   - 계획에 없지만 변경된 파일 식별
4. **최근 커밋 스타일 확인**: `git log`로 최근 커밋 메시지 스타일을 파악합니다.

### Phase 2: 사용자 보고 및 결정

1. **변경 내역 요약 보고**:
   - 업무 제목 및 요약
   - 변경된 파일 목록 (업무계획서 기반 분류)
   - 완료된 Todo 항목과 관련 파일 매핑
   - 계획 외 변경 사항이 있으면 별도 표시
2. **사용자 결정 요청**:
   - 커밋 범위 (전체/선택적 파일)
   - 커밋 메시지 수정 여부
   - 단일 커밋 vs 분리 커밋

### Phase 3: 커밋 수행

1. 사용자가 결정한 범위의 파일을 `git add`로 스테이징합니다.
2. 업무계획서 기반으로 생성한 커밋 메시지로 커밋합니다.
   - 커밋 메시지 형식:
     ```
     {업무 제목 요약}

     - {완료된 주요 변경 사항 1}
     - {완료된 주요 변경 사항 2}
     ...

     Task: {업무계획서 파일명}
     Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
     ```
3. 커밋 결과를 사용자에게 보고합니다.

## 중요 규칙

- **커밋 전 반드시 사용자 승인** 필요 (자동 커밋 금지)
- 커밋 메시지는 사용자가 수정할 수 있도록 **제안 형태**로 제시합니다.
- `.claude/tasks/` 폴더의 문서 파일들은 커밋에 **포함하지 않습니다** (업무 관리용 파일이므로).
- push는 사용자가 **명시적으로 요청한 경우에만** 수행합니다.
- 커밋 메시지는 반드시 HEREDOC 형식으로 전달합니다.
