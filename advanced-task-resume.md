# 고급업무재개

이전 세션에서 진행하던 업무를 문서 기반으로 재개하는 스킬입니다.

## 사용법

`/advanced-task-resume [파일명 또는 키워드]`

- 인자가 없으면: `.claude/tasks/` 폴더의 "진행중" 상태 업무를 자동 탐색
- 파일명 지정: 해당 업무계획서를 직접 로드
- 키워드 지정: 키워드로 업무계획서 검색

## 수행 절차

### Phase 1: 문서 로드 및 컨텍스트 복원

1. **업무 탐색**: `.claude/tasks/` 폴더에서 관련 문서를 찾습니다.
   - 업무계획서(`task_*.md`) 로드
   - TodoList(`todo_*.md`) 로드
   - "진행중" 상태인 업무가 여러 개면 사용자에게 선택 요청

2. **컨텍스트 복원**: 업무계획서를 읽고 맥락을 파악합니다.
   - 업무 요약, 분석 결과, 구현 계획 확인
   - 관련 파일들의 현재 상태 확인 (실제 파일을 Read로 읽어서 문서 내용과 비교)
   - 코드 스니펫이 현재 코드와 일치하는지 검증

2-1. **참조 이미지 복원**: 업무계획서에 "참조 이미지" 섹션이 있는 경우:
   - 기록된 이미지 파일들의 존재 여부를 확인합니다 (파일 경로로 접근 가능한지).
   - 존재하는 이미지를 Read 도구로 열어서 시각적 컨텍스트를 복원합니다.
   - 이미지의 텍스트 설명과 실제 이미지 내용을 대조하여 맥락을 파악합니다.
   - 누락된 이미지가 있으면 Phase 2에서 사용자에게 알립니다.

3. **진행 상황 확인**: TodoList에서 완료/미완료 항목을 파악합니다.
   - 완료된 항목(`[x]`) 확인
   - 미완료 항목(`[ ]`)부터 재개할 준비

### Phase 2: 사용자 브리핑

사용자에게 다음을 보고합니다:
- 업무 요약
- 지금까지 완료된 항목
- 남은 작업 목록
- 코드 변경 사항이 문서와 다른 부분이 있다면 알림
- 참조 이미지 로드 결과 (정상 로드/누락 파일 목록)

사용자 승인 후 Phase 3으로 진행합니다.

### Phase 3: 업무 재개

1. **TodoWrite 도구**에 남은 항목들을 등록합니다.
2. 미완료 Todo 항목을 **순서대로** 수행합니다.
3. 각 Todo 완료 시:
   - TodoList 파일(`todo_*.md`)에서 해당 항목을 `[x]`로 체크
   - "완료 기록" 섹션에 변경 내용 메모 추가
   - TodoWrite 도구에서도 completed로 마킹
4. 모든 Todo 완료 시 → **Phase 4로 진행**

### Phase 4: 완료 검증

모든 구현 작업이 끝나면, 아래 절차로 결과를 검증합니다.

#### 4-1. AI 자동 검증

다음 항목들을 AI가 자동으로 수행하고 결과를 보고합니다:

1. **MCP 컴파일 체크** (CLAUDE.md "Build Validation" 절차 준수):
   - `read_console(action="clear")` → `refresh_unity(mode="force", compile="request", scope="all", wait_for_ready=true)` → `read_console(types=["error"], filter_text="error CS", format="detailed")`
   - 에러가 있으면 수정 후 재검증 (Phase 3으로 돌아가 수정)
   - **컴파일 성공할 때까지 Phase 4를 완료하지 않습니다.**

2. **계획 대비 변경 대조**:
   - `git diff`로 실제 변경된 파일 목록 확인
   - 업무계획서의 "변경 사항 목록"과 대조
   - 누락된 변경이나 계획에 없는 변경이 있으면 보고

3. **코드 품질 검사**:
   - 변경된 C# 파일에 대해 `validate_script` 실행
   - TODO/FIXME/HACK 주석이 남아있는지 확인
   - 구독 누락 검사: UniRx `.Subscribe()`에 `.AddTo()` 누락 여부
   - null 참조 위험이 명백한 코드가 없는지 확인

4. **TodoList 완전성 확인**:
   - `todo_*.md`의 모든 항목이 `[x]` 체크되었는지 확인
   - 미완료 항목이 있으면 이유를 정리

#### 4-2. 사용자 확인 체크리스트 생성

변경된 파일과 작업 내용을 분석하여, **사람이 직접 확인해야 하는 항목**을 체크리스트로 생성합니다.

다음 기준으로 자동 분류합니다:

| 변경 유형 | 확인 항목 예시 |
|----------|--------------|
| UI 프리팹 (Panel_*, Popup_*, Piece_*) | "Unity에서 해당 화면을 열고 레이아웃/정렬/텍스트가 정상인지 확인" |
| 이펙트/파티클 (fx_*, Effect/) | "Play 모드에서 이펙트 재생 확인 (타이밍, 위치, 색상)" |
| Spine 애니메이션 | "Play 모드에서 스파인 연출이 정상 재생되는지 확인" |
| DOTween/VisualAction 연출 | "Play 모드에서 트윈 연출이 의도대로 동작하는지 확인" |
| 사운드/진동 | "실기기에서 사운드/햅틱 재생 확인" |
| 스크롤/리스트 UI | "아이템 수가 많을 때 스크롤이 정상 동작하는지 확인" |
| 해상도/레이아웃 | "다양한 해상도에서 UI가 깨지지 않는지 확인" |
| 서버 통신 (Protocol/API) | "개발 서버 연결 상태에서 정상 동작 확인" |
| 로컬 데이터 저장 | "앱 재시작 후 데이터가 유지되는지 확인" |
| 테이블 데이터 참조 | "테이블 값 변경 시 UI/로직에 정상 반영되는지 확인" |
| 플랫폼 분기 (#if UNITY_IOS 등) | "iOS/Android 각각에서 동작 확인" |

**체크리스트 형식** (업무계획서와 TodoList에 추가):

```markdown
## 6. 완료 검증

### AI 자동 검증 결과
- ✅ 컴파일: 성공 (에러 0개)
- ✅ 계획 대비 변경: 일치
- ✅ 코드 품질: 이상 없음
- ✅ TodoList: 전체 완료

### 사용자 확인 체크리스트
> 아래 항목은 Unity Editor 또는 실기기에서 직접 확인이 필요합니다.

- [ ] {확인 항목 1}: {구체적으로 무엇을 어디서 확인해야 하는지}
- [ ] {확인 항목 2}: {구체적으로 무엇을 어디서 확인해야 하는지}
...
```

#### 4-3. 최종 보고

1. AI 자동 검증 결과를 요약하여 보고합니다.
2. 사용자 확인 체크리스트를 표시합니다.
3. 업무계획서에 "6. 완료 검증" 섹션을 추가합니다.
4. TodoList에 "검증 체크리스트" 섹션을 추가합니다.
5. 업무계획서의 상태를 **"검증 대기"**로 변경합니다.
   - 사용자가 체크리스트를 모두 확인한 후 "완료"로 최종 변경합니다.
   - 사용자가 확인 중 문제를 발견하면 수정 작업을 추가로 진행합니다.

### 중간 요구사항 변경 처리

업무 수행 도중 사용자가 **요구사항 변경, 추가, 삭제**를 요청하면 다음을 **반드시** 수행합니다:

1. **업무계획서(`task_*.md`) 업데이트**:
   - "업무 요약" 섹션에 변경된 요구사항 반영
   - "구현 계획" 섹션의 접근 방식, 변경 사항 목록 수정
   - 변경 이력을 문서 하단에 기록 (예: `### 변경 이력` → `- {날짜}: {변경 내용 요약}`)
   - 변경으로 인해 새로운 관련 파일이나 주의사항이 생기면 해당 섹션도 업데이트

2. **TodoList(`todo_*.md`) 업데이트**:
   - 불필요해진 Todo 항목 삭제 또는 취소 표시 (`~취소됨~`)
   - 새로 필요한 Todo 항목 추가
   - 기존 항목의 내용이 변경되면 수정
   - "최종수정" 날짜 갱신

3. **TodoWrite 도구 동기화**:
   - TodoList 파일 변경에 맞춰 TodoWrite 도구의 항목도 함께 업데이트

4. **사용자 확인**:
   - 변경된 계획과 Todo를 사용자에게 요약 보고
   - 승인 후 변경된 계획에 따라 업무 계속 수행

> **핵심 원칙**: 문서는 항상 **현재 업무의 최신 상태**를 반영해야 합니다. 요구사항이 변경되었는데 문서가 이전 상태로 남아있으면, 세션 재개 시 혼란을 초래합니다.

## 중요 규칙

- 문서와 실제 코드 상태가 다를 수 있으므로, 반드시 실제 파일을 확인한 후 작업합니다.
- 이전 세션에서 부분적으로 완료된 작업이 있을 수 있으니, 완료 기록을 꼼꼼히 확인합니다.
- 문서에 기록된 계획이 현재 상황과 맞지 않으면, 계획을 수정하고 사용자에게 알립니다.
- 참조 이미지가 누락된 경우, 텍스트 설명을 기반으로 작업을 계속하되, 이미지 재제공이 필요한지 사용자에게 확인합니다.
